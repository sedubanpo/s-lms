강사 계정 요청

Auth, Firestore 계정 문서, Teachers 시트를 함께 갱신합니다.

계정 생성은 Firebase Auth, `users`, `userProfiles`, `userAppAccess`, `loginAliases`, Sheets `Teachers`가 같은 요청 ID로 처리되어야 합니다.

연동 설정

운영 배포 전 Firebase Cloud Functions 또는 Cloud Run 엔드포인트를 연결합니다.

API가 설정되지 않은 상태에서는 실제 저장소에 반영하지 않고 요청 payload만 로컬 대기열에 쌓습니다. 운영 저장은 Firebase 서버 API에서만 처리합니다.

서버 API 계약

구현해야 하는 단일 엔드포인트입니다.

action: accountManagementBatch요청 배열을 받아 Firebase 원본을 저장하고, Sheets 미러링 작업 상태와 rollbackHint를 반환합니다.
필수 응답 필드ok, requestId, committedTargets, failedTargets, auditLogId, rollbackPlan, dataVersion
보안관리자 세션 검증, CSRF성 토큰, 운영자 uid 기록, 서버측 중복 검사를 수행해야 합니다.
