학생 변경 요청

Firestore students 문서에 직접 저장할 payload를 만듭니다.

학생 생성/상태 변경은 Firestore `students`에 저장합니다. Sheets 미러링이 필요하면 pending action으로 남깁니다.

강사 계정 요청

Firestore 계정 문서, 로그인 alias, S-edu page 동기화 데이터를 갱신합니다.

Firebase Auth는 비밀번호 원문 조회를 지원하지 않습니다. 이 값은 관리자 인증 후 S-edu page `Teachers` 시트에서 불러온 운영용 비밀번호입니다.

S-LMS 관리자 세션이 있으면 S-edu page에 즉시 반영하고, 없으면 Sheets sync job으로 남깁니다.

No-Blaze 원칙

Functions와 Admin SDK 없이 릴리스합니다.

직접 저장관리자 로그인 후 Firestore rules가 허용하는 문서만 저장합니다.
후속 작업Auth 작업은 accountManagementRequests에 남기고, Sheets 작업은 S-LMS 세션이 있으면 즉시 반영합니다.
보안관리자 권한은 users/userAppAccess 문서와 Firestore rules로 확인합니다.
